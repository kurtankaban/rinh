# Домашнее задание: Внедрение системы управления проектно-сметной документацией в СГК

## 1. Генеральная цель проекта

Внедрение единой автоматизированной системы управления проектно-сметной документацией на платформе «1С: Предприятие 8» в компании «Стройгазконсалтинг», обеспечивающей полный цикл работы с документацией (получение, экспертиза, передача в производство, архивирование) для всех территориальных подразделений и удалённых объектов строительства.

## 3. Система управления проектом и план реализации

### Организационная структура проекта

| Роль | Ответственность |
|------|----------------|
| **Куратор проекта** (заместитель генерального директора СГК) | Стратегическое руководство, выделение ресурсов, решение эскалированных вопросов |
| **Руководитель проекта** (от СГК) | Оперативное управление, контроль сроков и бюджета, координация команды |
| **Руководитель проекта** (от подрядчика — партнёр 1С) | Техническая реализация, настройка системы, интеграция |
| **Рабочая группа** | Специалисты СГК (ПТО, сметный отдел, IT-служба, архив) + консультанты подрядчика |
| **Группа тестирования** | Ключевые пользователи от каждого подразделения |

### Методология

Проект реализуется по **каскадной модели (waterfall)** с элементами итеративного подхода на этапе настройки, что типично для внедрения 1С.

### Этапы реализации

| Этап | Содержание | Срок |
|------|-----------|------|
| 1. Обследование | Анализ текущих бизнес-процессов работы с ПСД, сбор требований, обследование IT-инфраструктуры | Месяцы 1–2 |
| 2. Проектирование | Разработка технического задания, проектирование архитектуры системы, маршрутов документов, ролевой модели доступа | Месяцы 2–3 |
| 3. Разработка и настройка | Настройка/доработка конфигурации 1С, создание справочников, настройка маршрутов, интеграция с существующими системами | Месяцы 3–6 |
| 4. Тестирование | Функциональное тестирование, нагрузочное тестирование, приёмочное тестирование ключевыми пользователями | Месяцы 6–7 |
| 5. Обучение | Разработка инструкций, проведение обучающих семинаров для всех групп пользователей | Месяцы 7–8 |
| 6. Опытная эксплуатация | Запуск в пилотном подразделении, сбор обратной связи, устранение замечаний | Месяцы 8–9 |
| 7. Промышленная эксплуатация | Тиражирование на все подразделения и удалённые объекты | Месяцы 9–12 |
| 8. Сопровождение | Техническая поддержка, доработки по результатам эксплуатации | С месяца 12 — постоянно |

### Инструменты управления

- **Планирование:** диаграмма Ганта (MS Project / аналог) с контрольными точками (вехами)
- **Коммуникации:** еженедельные статус-совещания рабочей группы, ежемесячные отчёты куратору
- **Управление рисками:** реестр рисков с оценкой вероятности и влияния, план реагирования
- **Управление изменениями:** формализованная процедура запроса на изменение (CR), рассмотрение на управляющем комитете
- **Документирование:** протоколы совещаний, реестр требований, журнал проблем

## 4. Критерии успешной реализации проекта

### Обязательные критерии (проект считается успешным при выполнении всех)

1. **Функциональная полнота:** система обеспечивает полный цикл работы с ПСД — приём документации от заказчиков и проектных институтов, рассмотрение и экспертиза, передача подрядным организациям, архивное хранение.

2. **Охват пользователей:** все подразделения компании (включая удалённые объекты строительства) подключены к системе и работают в ней.

3. **Единая база данных:** создана и поддерживается в актуальном состоянии единая база проектно-сметной документации с контролем версий.

4. **Разграничение доступа:** реализована ролевая модель — каждый пользователь имеет доступ только к тем функциям и документам, которые соответствуют его полномочиям.

5. **Соблюдение сроков и бюджета:** проект завершён в рамках утверждённого бюджета с отклонением по срокам не более 15%.

### Количественные показатели эффективности (оцениваются через 6 месяцев после запуска)

| Показатель | Целевое значение |
|------------|-----------------|
| Среднее время прохождения документа по маршруту | ≤ 5 рабочих дней (было ~10) |
| Доля документов, обрабатываемых в системе (а не вручную/по email) | ≥ 95% |
| Количество инцидентов работы с неактуальной версией документации | Снижение на 90% |
| Время поиска документа в архиве | ≤ 2 минуты (было — от 30 минут до нескольких часов) |
| Удовлетворённость пользователей (по результатам опроса) | ≥ 4 из 5 баллов |
| Доля обученных сотрудников | ≥ 95% целевой аудитории |
